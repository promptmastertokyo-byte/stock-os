# 自動化アーキテクチャ v1.0

## 判断基準

MCPは「AIが対話的に複数のデータ源や操作を選ぶ必要がある機能」に限定する。
決定的で反復可能な処理は通常スクリプトにし、テストとGitHub Actionsで再現できるようにする。

## 機能の分離

|機能|方式|理由|
|---|---|---|
|企業・証券コード検索|MCP候補|曖昧な企業名解決と対話的選択が必要|
|IR資料検索・候補提示|MCP候補|複数ページから最新・適切な資料を選ぶ|
|資料本文の要約・論点抽出|MCP候補|非構造データの意味理解が必要|
|出典付き分析の下書き|MCP候補|複数情報の統合と不確実性判断が必要|
|株価・指標の定時取得|通常スクリプト|入力と出力が定型で再現しやすい|
|日付・必須見出し検査|通常スクリプト|決定的な静的検査|
|配点合計・上限検査|通常スクリプト|機械的に判定可能|
|決算予定・再評価日の計算|通常スクリプト|日付計算としてテスト可能|
|GitHub Issue起票|GitHub Actions/API|実行履歴と権限をGitHub側で管理|
|売買判断・注文|自動化しない|人間の承認境界を維持する|

## 処理境界

1. 取得層が生データと出典メタデータを返す。
2. 検証層が型、期間、単位、欠損を確認する。
3. 分析層が根拠を先に記入し、その後に採点する。
4. レビュー層が反証、期間混同、循環補正、出典一致を確認する。
5. 人間が最終判断し、承認後のみ判断欄を確定する。

## MCP化の着手条件

次をすべて満たすまではMCPを作らない。

- 同じ手作業を3回以上実行し、入出力が安定した
- 通常スクリプトや既存コネクタで代替できない
- 認証情報をリポジトリ外で安全に管理できる
- 失敗時に部分的な古いデータを「最新」と誤認しない設計がある
- 呼び出し回数、レート制限、取得日時を記録できる
