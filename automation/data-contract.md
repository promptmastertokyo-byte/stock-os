# データ取得契約 v1.0

## 目的

情報源ごとの役割と取得項目を固定し、企業ごと・実行者ごとの分析差を減らす。
すべての数値に `as_of`（対象時点）、`retrieved_at`（取得日）、`source_url` を持たせる。

## 情報源の優先順位

1. 企業IR（決算短信、決算説明資料、有価証券報告書、適時開示）
2. EDINET・TDnetなどの公的開示
3. 取引所・指数提供者
4. Yahoo!ファイナンス等の二次情報

二次情報は株価・時価総額・指標の入口に使えるが、業績・財務・会社計画は原則として一次情報で照合する。

## 共通メタデータ

|項目|必須|説明|
|---|---|---|
|code|必須|証券コード。英字を含むコードは大文字で保存|
|company_name|必須|開示上の正式名称|
|market|必須|市場区分|
|currency|必須|原則JPY。外貨の場合は通貨を明記|
|as_of|必須|数値の対象日または対象期間|
|retrieved_at|必須|取得日（YYYY-MM-DD）|
|source_name|必須|資料名またはページ名|
|source_url|必須|直接確認できるURL|
|source_type|必須|`primary` / `official` / `secondary`|
|status|必須|`confirmed` / `provisional` / `unverified`|

## Yahoo!ファイナンス等から取得する市場データ

|項目|取得タイミング|注意|
|---|---|---|
|株価・前日比|分析実行時|場中か終値かを明記|
|時価総額|分析実行時|株式数の基準日差に注意|
|PER・PBR|分析実行時|実績／予想の別を明記|
|配当利回り|分析実行時|会社予想か実績かを明記|
|年初来高値・安値|分析実行時|過熱感の補助情報に限定|
|出来高・売買代金|分析実行時|需給評価の補助情報に限定|

## 企業IRから取得する一次データ

|区分|最低取得項目|原則期間|
|---|---|---|
|業績|売上高、営業利益、純利益、EPS、各成長率|実績3〜5年＋会社予想|
|収益性|営業利益率、ROE、可能ならROIC|実績3〜5年|
|財務|現金同等物、有利子負債、自己資本比率、発行済株式数|直近＋前期|
|CF|営業CF、投資CF、FCF|実績3〜5年|
|還元|配当、配当性向、自社株買い・消却|実績3年＋計画|
|事業|セグメント売上・利益、地域、主要顧客依存|直近通期|
|計画|会社予想、中期計画、設備投資、研究開発|最新開示|
|リスク|為替感応度、規制、訴訟、希薄化、重要な後発事象|最新開示|

## 欠損・競合時のルール

- 一次情報が取得できなければ `unverified` とし、未確認項目へ移す。
- 複数ソースで値が違う場合は定義・対象期間・単位を確認し、一次情報を採用する。
- 推定値は元データと計算式を併記し、確認済み数値と同じ欄で断定しない。
- 市場コンセンサスと会社計画は別フィールドとして扱う。
- 株式分割がある場合は、株価・EPS・配当の調整前後を混在させない。

## 保存単位

分析Markdownは読みやすい要約とし、将来構造化データを導入する場合も上記フィールド名を維持する。
取得したHTMLやPDF全文は再配布せず、URL・資料名・対象期間・確認したページを保存する。
